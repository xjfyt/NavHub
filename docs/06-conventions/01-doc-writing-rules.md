# 文档编写规范

> **对应代码**：`docs/` 全目录
> **维护提示**：修改文档规范时同步更新本文档。

## 1、文档结构规范

### 1.1 目录编号

所有文档目录使用两位数编号前缀：

```
01-overview/
02-getting-started/
03-config/
04-modules/
05-maintenance/
06-conventions/
07-deployment/
```

### 1.2 文件编号

同一目录下的文件使用两位数编号前缀：

```
01-introduction.md
02-architecture.md
03-tech-stack.md
```

### 1.3 子文档拆分规则

当单个文档超过 **300 行**或覆盖 **5 个以上独立主题**时，应拆分为子目录：

```
04-modules/                    # 模块级目录
├── 01-auth/                   # 认证模块
│   ├── 01-overview.md         # 总览（必有）
│   ├── 02-permissions.md      # 权限管理
│   └── ...
├── 02-database/               # 数据库模块
└── ...
```

## 2、文件头元数据

每个文档 **必须** 以 blockquote 元数据开头：

```markdown
> **对应代码**：`backend/src/auth/mod.rs`
> **维护提示**：修改认证逻辑时同步更新本文档。
```

### 2.1 对应代码

- 列出本文档描述的所有源文件路径
- 使用相对路径（从 docs/ 到 backend/src/）
- 多个路径用顿号（、）分隔

### 2.2 维护提示

- 明确说明何时需要更新本文档
- 使用"修改/新增/移除 ... 时同步更新本文档"句式

## 3、标题层级

| 层级 | 格式 | 示例 |
|------|------|------|
| H1 `#` | 中文序数 + 顿号 | `一、项目简介` |
| H2 `##` | 阿拉伯数字 + 顿号 | `1、概述` |
| H3 `###` | 括号数字 | `（1）启动流程` |
| H4 `####` | 圆圈数字 | `① 第一步` |
| H5+ | 使用列表 | 不再使用标题 |

## 4、文件尾导航

每个文档 **必须** 以导航链接结尾：

```markdown
---
- 上一篇：[xx-xxx.md](./xx-xxx.md)
- 下一篇：[yy-yyy.md](./yy-yyy.md)
- 返回索引：[docs/README.md](../README.md)
```

- 首篇的"上一篇"写 `（无，这是首篇）`
- 末篇的"下一篇"写 `（无，这是末篇）`
- 索引路径根据文档深度调整（`../`、`../../` 等）

## 5、交叉引用规范

### 5.1 文档间引用

使用相对路径 + 章节锚点：

```markdown
详见 [01-auth/01-overview.md](./04-modules/01-auth/01-overview.md)
详见 [01-config-file.md §2.4](./03-config/01-config-file.md)
```

### 5.2 代码引用

使用相对路径指向源文件：

```markdown
[mod.rs](../../backend/src/auth/mod.rs)
[`hash_password`](../../backend/src/auth/password.rs)
```

### 5.3 章节锚点

使用 `§` 符号引用章节：

```markdown
§三、（2）
§2.4
```

## 6、内容格式

### 6.1 表格

优先使用表格展示：

- API 端点列表
- 配置字段说明
- 状态枚举
- 功能对比

### 6.2 代码块

- Rust 代码使用 ` ```rust `
- TypeScript 代码使用 ` ```typescript `
- Shell 命令使用 ` ```bash `
- 配置文件使用对应语言标记

### 6.3 ASCII 图表

使用 ASCII art 绘制架构图、流程图：

```
┌─────────┐     ┌─────────┐
│ Module A │ ──→ │ Module B │
└─────────┘     └─────────┘
```

## 7、代码 ↔ 文档双向连接

### 7.1 代码变更 → 文档更新

| 代码变更 | 文档操作 |
|----------|----------|
| 新增/重命名模块 | 更新模块总览，新增对应子文档 |
| 修改核心逻辑 | 更新对应模块文档 |
| 新增/修改 API | 更新 API 文档 |
| 修改配置常量 | 更新配置文档 |
| 修改数据库 schema | 更新数据库文档 |

### 7.2 文档审查

- 每次 PR 审查时检查文档是否需要更新
- 文档过期比没有文档更危险（误导开发者）

## 8、文档拆分决策树

```
文档超过 300 行？
├── 是 → 拆分为子目录
└── 否 → 保持单文件

覆盖 5+ 独立主题？
├── 是 → 按主题拆分子文档
└── 否 → 保持单文件

模块有 5+ 个源文件？
├── 是 → 创建模块子目录 + overview.md
└── 否 → 在模块总览中简要描述
```
