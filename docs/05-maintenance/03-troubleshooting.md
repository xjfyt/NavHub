# 常见问题排查

> **对应代码**：全项目
> **维护提示**：发现新的常见问题时补充本文档。

## 1、构建问题

### 1.1 Rust 编译失败

**问题**：`error[E0463]: can't find crate`

**解决**：检查 `backend/rust-toolchain.toml` 确保 Rust 版本为 `1.95.0`：

```bash
rustup show
```

### 1.2 数据库连接失败

**问题**：`connection refused` 或 `timeout`

**排查**：
1. 检查 PostgreSQL 服务是否运行
2. 检查 `config.toml` 中的数据库配置
3. 检查网络连接和防火墙设置

### 1.3 Redis 连接失败

**问题**：`connection refused` 或 `timeout`

**排查**：
1. 检查 Redis 服务是否运行
2. 检查 `config.toml` 中的 Redis 配置
3. 检查 Redis 密码是否正确

### 1.4 前端构建失败

**问题**：Vite 构建失败

**解决**：确保 Node.js >= 20：

```bash
node --version
```

## 2、运行时问题

### 2.1 登录失败

**问题**：`401 Unauthorized` 或 `403 Forbidden`

**排查**：
1. 检查用户名和密码是否正确
2. 检查 SSO 配置是否正确
3. 检查用户角色和权限

### 2.2 会话失效

**问题**：频繁需要重新登录

**排查**：
1. 检查 Cookie 设置是否正确
2. 检查 Redis 连接是否正常
3. 检查会话 TTL 配置

### 2.3 文件上传失败

**问题**：`413 Payload Too Large` 或上传失败

**排查**：
1. 检查上传文件大小限制（默认 2MB）
2. 检查 S3 存储配置
3. 检查网络连接

### 2.4 图标加载失败

**问题**：图标显示为空白或加载失败

**排查**：
1. 检查图标 URL 是否正确
2. 检查网络连接
3. 检查 S3 存储配置

## 3、前端问题

### 3.1 热更新不生效

**解决**：重启 `npm run dev`

### 3.2 类型错误

**解决**：运行 `npm run lint` 检查

### 3.3 组件渲染错误

**排查**：
1. 检查浏览器控制台错误信息
2. 检查组件 props 是否正确
3. 检查状态管理逻辑

## 4、部署问题

### 4.1 Docker 容器启动失败

**排查**：
1. 检查 Docker 日志：`docker logs navhub`
2. 检查配置文件挂载
3. 检查端口映射

### 4.2 数据库迁移失败

**排查**：
1. 检查迁移文件语法
2. 检查数据库权限
3. 检查迁移顺序

### 4.3 S3 存储连接失败

**排查**：
1. 检查 S3 配置
2. 检查网络连接
3. 检查访问密钥和秘密密钥

## 5、性能问题

### 5.1 页面加载缓慢

**排查**：
1. 检查网络连接
2. 检查图片大小和格式
3. 检查 JavaScript 包大小

### 5.2 API 响应缓慢

**排查**：
1. 检查数据库查询性能
2. 检查 Redis 缓存命中率
3. 检查服务器资源使用情况

## 6、安全问题

### 6.1 密码泄露

**解决**：
1. 立即修改密码
2. 检查日志是否有异常登录
3. 启用双因素认证（如果支持）

### 6.2 未授权访问

**排查**：
1. 检查用户权限设置
2. 检查 API 访问控制
3. 检查日志中的异常访问

## 7、日志查看

### 7.1 后端日志

```bash
# Docker 容器日志
docker logs navhub

# 本地开发日志
cargo run
```

### 7.2 前端日志

打开浏览器开发者工具，查看 Console 和 Network 面板。
