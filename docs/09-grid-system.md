# 方格排版系统 (Grid System) 架构纪要

NavHub 的核心体验依赖于一套精准的网格拖拽排版体系。在早期实现中，由于响应式计算与 CSS 视觉样式脱节，导致了大量“幽灵墙”、“视觉间距错觉”和“空气碰撞”的 Bug。本章详细记录了重写后的 **「物理级光栅网格系统（Absolute Geometry Mesh）」** 的底层逻辑和数学原理。

## 1. 核心挑战：视觉体积与物理网格框的悖论

基于 `react-grid-layout` 的底层算法，网格距离公式为：
**占位物理宽度 = N个网格宽度 + (N-1)个物理间距**

在旧架构中，基础的单元格大小 (`cols=8`, 小图标占 `wSpan:1`) 遇到了以下致命悖论：
- 图标渲染大小由前端固定（如 `width: 56px`）。如果父级分配给它的物理格子有 `80px`（由于大屏拉伸），那么内部就会产生 `24px` 的伪留白。
- 当两个图块相距 **1 个格子** 时，1 个物理格子的留白（例如 80px），如果在视觉上恰好略大于 56px，用户的常识直觉就会认为这个“视觉洞”能够再填入一个图标。
- 但底层的代数引擎检查时会报错：因为要拖放的图标不仅需要 56 像素的空间，还需要“完整的对应网格占位（N个列）”。从而引发明明肉眼看到放得下，但死活拖不进去或导致相邻组件乱飞的现象。

---

## 2. 解决方案一：100% 绝对物理边界对齐 (Absolute Box Padding)

为消灭上述“几何学错觉”，新架构规定：**所见即所得，DOM 元素的视觉填充面积必须 100% 等于其底层的碰撞盒（Bounding Box）。**

1. **废绝内部留白填充**：CSS 层面不再对图标使用任何带固定宽度的居中定位。直接覆盖样式为：
   ```css
   .tile.sq .tile-icon { 
     width: 100%; 
     height: 100%; 
     border-radius: 20%; /* 放弃固定圆角，采用受容器驱动的百分比过渡（iOS风格）*/ 
   }
   ```
2. **文本外置 (Absolute Overlay)**：为了让图标能够撑满100%几何格子，文字标签 `.tile-label` 被改为了 `position: absolute; bottom: -24px;`，这样它们就可以静静躺在相邻组件物理属性外围的 `margin` 空隙中，从而彻底解决容器撑破问题。

---

## 3. 解决方案二：严格锁定 16px 的苹果风静态间隙 (Static Margin Lock)

为了保持 iOS 桌面那令人舒适的视觉一致性，不再把“间隙”（Margin）交给 `react-grid-layout` 响应式算法胡乱处理，而是采取“数学锁死”机制。
在新系统下，不仅强制了 `margin={[16, 16]}` 的静态物理屏障，同时强制去除了 RGL 断点系统内置的列数改变：

```tsx
// NavView.tsx 核心 ResizeObserver
const colsSetting = tweaks.gridCols || 32; // 默认网格光栅密度为 32列
const M = 16;
const totalGapPixels = M * (colsSetting - 1);
const X = Math.max((w - totalGapPixels) / colsSetting, 10); // 反向算出每个小格子的纯净宽度 X
setMeshUnit(X);
```

**效果：**
由于数学严丝合缝，当 `X=22px` 时，标准 2 宫格组件的宽度是 `2*X + 16 = 60px`（极其接近 iOS 图标真实大小）。而此组件移动一小格（Delta=1），恰好对应着拖动了刚好足够判断不粘滞的最佳间隙。此后再也没有“只跨半个图标”导致网格错乱碎裂的缝隙了！

---

## 4. 解决方案三：全局全屏截流与“迷路元素打捞”机制

为了对齐桌面操作系统的“全视界体验”，我们剥离了所有原生的网页滚动条 `overflow-y: scroll`，使得画布**严格限定在屏幕可视视口 (Viewport) 的最底侧**。也就是在样式中强加了 `overflow: hidden` 到 `.content`。

此时，旧有那些放置在画布深处坐标的组件必然无法被用户发现。对此设计了 **智能自愈截流算法**：

```tsx
// 在每一次重新渲染排版前，推演所有元素当前所属的底边界线
const vRows = Math.floor((h + M) / (X + M)); // 计算当前显示器高最多呈现多少行

//... 迭代组件列表：如果查出某组件 Y值+自身高度 已超过了当前最大的 vRows
if (gY !== null && (gY + hSpan > maxRows || gY < 0)) {
   gY = null; // 判定该元素被“截肢沉入海底”
   gX = null;
}
```

利用 RGL `compactType=null` 对“未指派坐标 (`gX/gY=null`)” 实施自动安全顶配排列机制，所有飞出屏幕的图标瞬间被自动召回到头部空白坑位之中，彻底解决了调整窗口大小或全屏切换时的坐标轴失禁窘境。

总结起来，这套方格系统的终极状态可以浓缩为：**视觉所及边缘即碰撞体积死线，没有任何弹性与虚空。**
