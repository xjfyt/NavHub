# 一、图标管理

> **对应代码**：`backend/src/handlers/icons.rs`、`frontend/src/components/NavView.tsx`、`frontend/src/components/IconTile.tsx`
> **维护提示**：修改图标 CRUD 逻辑、样式系统或拖拽排序时同步更新本文档。

NavHub 的图标管理模块是导航页的核心功能，负责图标的创建、编辑、删除、排序以及文件夹合并等操作。每个图标代表一个快捷入口，支持丰富的样式配置和灵活的拖拽排序。

## 1、图标数据模型

### 1.1 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 图标唯一标识 |
| `group_id` | UUID | 所属分类 ID |
| `name` | String | 图标名称（必填） |
| `url` | String? | 跳转链接 |
| `sub` | String? | 副标题 |
| `title` | String? | 鼠标悬停标题 |
| `cta` | String? | 行动号召文案 |
| `size` | IconSize | 图标尺寸 |
| `letter` | String? | 字母标识（无图片时显示） |
| `color` | i32 | 图标背景色（ARGB 整数） |
| `image_url` | String? | 图片 URL |
| `image_style` | Enum | 图片样式：`framed` / `plain` |
| `image_radius` | Enum | 图片圆角：`rounded` / `square` |
| `is_folder` | bool | 是否为文件夹图标 |
| `iframe_preview` | bool | 是否启用 iframe 预览 |
| `sort_order` | i32 | 排序序号 |
| `grid_x` / `grid_y` | i32? | 网格坐标（可选，备用） |
| `font_size` | Enum | 字体大小：`sm` / `md` / `lg` |
| `text_align` | Enum | 文本对齐：`left` / `center` / `right` |

### 1.2 图标尺寸规格

```typescript
type IconSize = "sq"       // 正方形（默认 1×1）
              | "pill-size" // 胶囊形（3×1）
              | "circle-size" // 圆形
              | "lg"       // 大号（2×2）
              | "lg-4"     // 4格宽（已废弃）
              | "lg-9";    // 9格宽（已废弃）
```

尺寸与网格单元的对应关系：

| IconSize | 网格跨度（w×h） | 适用场景 |
|----------|----------------|----------|
| `sq` | 1×1 | 普通快捷方式 |
| `pill-size` | 3×1 | 横向胶囊（如搜索栏） |
| `lg` | 2×2 | 大号图标（强调展示） |
| 文件夹 | 1×1 或 2×2 | 根据内含图标数量 |

## 2、CRUD 操作

### 2.1 创建图标

**API 端点**：`POST /api/icons`

**请求流程**：

```
┌──────────┐    POST /api/icons    ┌──────────┐    INSERT INTO icons    ┌──────────┐
│ Frontend  │ ──────────────────→ │  Handler  │ ────────────────────→ │ Database │
│ (Shell)   │    JSON Body        │ icons.rs  │    RETURNING *         │ (PgSQL)  │
└──────────┘                      └──────────┘                        └──────────┘
     ↑                                 │
     │         201 + IconView           │
     └─────────────────────────────────┘
```

**后端逻辑**（[icons.rs#L23-L89](../../backend/src/handlers/icons.rs)）：

1. 验证用户非 Guest 角色
2. 加载目标分类并校验写权限（`group_writable_by`）
3. 插入数据库，`sort_order` 自动取该分类下最大值 + 1
4. 记录审计日志
5. 返回完整的 `IconView`（含 `folder_items`）

**代码示例**：

```rust
// backend/src/handlers/icons.rs - 创建图标
pub async fn create(
    State(state): State<Arc<AppState>>,
    Extension(user): Extension<SessionUser>,
    Json(body): Json<IconCreate>,
) -> AppResult<(StatusCode, Json<IconView>)> {
    let g = load_group(&state, body.group_id).await?;
    if !util::group_writable_by(&g, &user) {
        return Err(AppError::Forbidden("not_owner"));
    }
    let icon: Icon = sqlx::query_as(
        "INSERT INTO icons (...) VALUES (...) RETURNING *"
    )
    .bind(body.group_id)
    // ... 其他字段绑定
    .fetch_one(&state.pg)
    .await?;
    Ok((StatusCode::CREATED, Json(view)))
}
```

### 2.2 更新图标

**API 端点**：`PATCH /api/icons/:id`

**关键逻辑**：
- 使用 `COALESCE` 实现部分更新（只传需要修改的字段）
- 如果修改 `group_id`（跨分类移动），额外校验目标分类的写权限
- 更新后重新加载 `IconView`（包含文件夹子项）

### 2.3 删除图标

**API 端点**：`DELETE /api/icons/:id`

**特殊处理**：
- 删除文件夹图标时，`folder_items` 表中的子项通过外键级联删除
- 删除操作记录审计日志

### 2.4 图标排序

**API 端点**：`POST /api/icons/reorder`

**请求体**：

```typescript
interface IconReorderRequest {
  groupId: string;    // 分类 ID
  order: string[];    // 图标 ID 数组，按新顺序排列
}
```

**实现方式**：在事务中逐条更新 `sort_order` 字段。

## 3、样式系统

### 3.1 图片样式

| 样式 | 说明 | CSS 类名 |
|------|------|----------|
| `framed` | 带边框/阴影的卡片风格 | `.icon-img-framed` |
| `plain` | 无边框透明背景 | `.icon-img-plain` |

### 3.2 图片圆角

| 圆角 | 说明 | CSS 效果 |
|------|------|----------|
| `rounded` | 圆角矩形 | `border-radius: 22%` |
| `square` | 直角矩形 | `border-radius: 4px` |

### 3.3 颜色系统

图标颜色以 ARGB 整数存储，前端解析后用于：
- 无图片时的背景色
- 字母标识的背景色
- 渐变效果的起始色

```typescript
// 前端颜色解析示例
function intToHex(color: number): string {
  return '#' + (color & 0xFFFFFF).toString(16).padStart(6, '0');
}
```

## 4、文件夹功能

### 4.1 合并为文件夹

**API 端点**：`POST /api/icons/:sourceId/merge-into/:targetId`

**合并流程**：

```
拖拽图标 A 到图标 B 上方
         │
         ▼
    ┌─────────┐
    │ A 与 B   │
    │ 重叠判定  │──→ 未达阈值 → 取消
    └─────────┘
         │ 达到阈值
         ▼
    ┌─────────┐
    │ 停留判定  │──→ 停留 < 800ms → 取消
    │ (800ms)  │
    └─────────┘
         │ 停留 ≥ 800ms
         ▼
    POST /api/icons/A/merge-into/B
         │
         ▼
    ┌─────────────────────────┐
    │ B.is_folder == false ?   │
    │  → 提升 B 为文件夹       │
    │  → 将 A、B 原数据都加入   │
    │ B.is_folder == true ?    │
    │  → 将 A 追加到 B 文件夹   │
    └─────────────────────────┘
         │
         ▼
    删除原图标 A
```

**安全校验**：合并操作同时校验源图标和目标图标的分类写权限（SEC-2），防止跨租户数据窃取。

### 4.2 从文件夹提取

**API 端点**：`POST /api/icons/:folderId/extract-item/:itemId`

将文件夹中的子项提取为独立图标，返回更新后的文件夹和新图标。

### 4.3 文件夹内排序

**API 端点**：`POST /api/icons/:folderId/reorder-folder-items`

在事务中更新 `folder_items` 表的 `sort_order`。

## 5、拖拽排序

### 5.1 前端拖拽架构

NavHub 使用 `@dnd-kit` 实现拖拽排序，核心组件：

```
Shell (DndContext)
  ├── Sidebar (useDroppable: "group:<id>")
  └── NavView (SortableContext)
        ├── NavGridCell (useSortable) × N
        └── DragOverlay (拖拽预览)
```

**关键设计**：
- `DndContext` 提升到 Shell 层，同时覆盖侧边栏和网格
- 侧边栏分类按钮作为 Droppable，支持拖拽到其他分类
- 网格内部使用 `rectSortingStrategy` 自动计算位置

### 5.2 拖拽传感器

```typescript
// frontend/src/hooks/useNavDnd.ts
const sensors = useSensors(
  useSensor(MouseSensor, {
    activationConstraint: { distance: 5 }  // 移动 5px 后激活
  }),
  useSensor(TouchSensor, {
    activationConstraint: { delay: 200, tolerance: 5 }  // 长按 200ms 激活
  }),
);
```

### 5.3 碰撞检测策略

自定义碰撞检测，分两层优先级：

1. **分类 Droppable 优先**：使用 `pointerWithin` 检测指针是否真正落入侧边栏分类按钮
2. **网格元素回退**：在非分类元素中使用 `closestCenter` 选择最近的目标

### 5.4 排序持久化

拖拽松手后：

1. **分类内排序**：调用 `POST /api/groups/:id/reorder-items`
2. **跨分类移动**：调用 `PATCH /api/icons/:id`（修改 `group_id`）+ 目标分类排序
3. **文件夹合并**：调用 `POST /api/icons/:id/merge-into/:target`

## 6、前后端交互总结

| 操作 | 前端触发 | API 端点 | 后端处理 |
|------|----------|----------|----------|
| 创建图标 | 添加按钮 / 拖入 | `POST /api/icons` | 插入数据库 + 审计 |
| 编辑图标 | 右键菜单 → 编辑 | `PATCH /api/icons/:id` | COALESCE 部分更新 |
| 删除图标 | 右键菜单 → 删除 | `DELETE /api/icons/:id` | 删除 + 级联清理 |
| 排序图标 | 拖拽松手 | `POST /api/icons/reorder` | 事务内批量更新 |
| 合并文件夹 | 拖拽深压 | `POST /api/icons/:id/merge-into/:target` | 创建/追加文件夹 |
| 提取子项 | 拖出文件夹 | `POST /api/icons/:id/extract-item/:target` | 提取为独立图标 |
