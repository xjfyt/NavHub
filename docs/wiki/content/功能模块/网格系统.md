# 五、网格系统

> **对应代码**：`frontend/src/components/NavView.tsx`、`frontend/src/hooks/useNavDnd.ts`、`frontend/src/utils/navVirtualization.ts`
> **维护提示**：修改网格布局、拖拽机制或响应式设计时同步更新本文档。

NavHub 的网格系统是导航页的核心布局引擎，负责将图标和小组件排列在 CSS Grid 容器中。它支持自动流式布局、拖拽排序、跨分类移动、文件夹合并等高级交互。

## 1、网格布局原理

### 1.1 CSS Grid 布局

NavHub 使用原生 CSS Grid 实现网格布局：

```css
.nav-grid {
  display: grid;
  grid-template-columns: repeat(var(--grid-cols, 6), 1fr);
  grid-auto-flow: row dense;
  gap: 16px;
}
```

**关键设计决策**：
- **`grid-auto-flow: row dense`**：浏览器自动计算位置，使用"紧密填充"算法避免空隙
- **不再保存 `gridX`/`gridY`**：每个元素只通过 `sortOrder` 决定顺序，位置由浏览器自动计算
- **响应式列数**：通过 CSS 变量 `--grid-cols` 控制列数（默认 6 列）

### 1.2 元素尺寸定义

网格元素的尺寸通过 `span` 类名控制：

```css
/* 基础单元 */
.nav-cell.w-1.h-1 { grid-column: span 1; grid-row: span 1; }
.nav-cell.w-2.h-1 { grid-column: span 2; grid-row: span 1; }
.nav-cell.w-2.h-2 { grid-column: span 2; grid-row: span 2; }
.nav-cell.w-3.h-1 { grid-column: span 3; grid-row: span 1; }
.nav-cell.w-4.h-2 { grid-column: span 4; grid-row: span 2; }
```

**图标尺寸映射**（[useNavDnd.ts#L43-L53](../../frontend/src/hooks/useNavDnd.ts)）：

| IconSize | 网格跨度 | CSS 类名 |
|----------|---------|---------|
| `sq` | 1×1 | `w-1 h-1` |
| `pill-size` | 3×1 | `w-3 h-1` |
| `lg` | 2×2 | `w-2 h-2` |
| 文件夹（普通） | 1×1 | `w-1 h-1` |
| 文件夹（大号） | 2×2 | `w-2 h-2` |

**小组件尺寸映射**（[useNavDnd.ts#L55-L63](../../frontend/src/hooks/useNavDnd.ts)）：

| WidgetSizeId | 网格跨度 | CSS 类名 |
|-------------|---------|---------|
| `small` | 3×1 | `w-3 h-1` |
| `medium` | 2×2 | `w-2 h-2` |
| `large` | 4×2 | `w-4 h-2` |

### 2.2 拖拽传感器

```typescript
// Mouse: 移动 5px 后激活（避免误触）
useSensor(MouseSensor, {
  activationConstraint: { distance: 5 }
})

// Touch: 长按 200ms 后激活（避免滑动误触）
useSensor(TouchSensor, {
  activationConstraint: { delay: 200, tolerance: 5 }
})
```

### 2.3 碰撞检测

NavHub 使用自定义的两层碰撞检测策略：

```typescript
const collisionDetection = useCallback((cdArgs) => {
  // 第 1 层：pointerWithin 检测指针是否落入侧边栏分类按钮
  const within = pointerWithin(cdArgs);
  const groupHit = within.find(
    (c) => parseGroupDroppableId(String(c.id)) !== null
  );
  if (groupHit) return [groupHit];  // 优先命中分类

  // 第 2 层：在网格元素中使用 closestCenter
  const gridContainers = cdArgs.droppableContainers.filter(
    (c) => parseGroupDroppableId(String(c.id)) === null
  );
  return closestCenter({ ...cdArgs, droppableContainers: gridContainers });
}, []);
```

**设计意图**：
- 分类 Droppable 优先，确保拖拽到侧边栏时准确命中
- 靠近侧边栏排序时不会误判为跨分类移动

### 2.4 拖拽操作类型

| 操作 | 触发条件 | 行为 |
|------|---------|------|
| **分类内排序** | 在网格内拖拽松手 | 更新 `sortOrder` |
| **跨分类移动** | 拖到侧边栏分类按钮 | 修改 `group_id` |
| **文件夹合并** | 深压到另一个图标 | 合并为文件夹 |
| **文件夹提取** | 从文件夹拖出 | 提取为独立图标 |

### 2.5 文件夹合并判定

文件夹合并使用"深压"交互，需要满足两个条件：

1. **重叠阈值**：拖拽元素与目标元素的重叠面积 ≥ 目标面积的 60%
2. **停留时间**：在目标上方停留 ≥ 800ms

```
拖拽图标 A 到图标 B
         │
         ▼
    重叠面积 ≥ 60% ?
    ┌─── NO ──→ 继续拖拽
    │
    YES
    │
    ▼
    停留 ≥ 800ms ?
    ┌─── NO ──→ 继续拖拽（显示合并提示）
    │
    YES
    │
    ▼
    触发合并：POST /api/icons/A/merge-into/B
```

## 3、性能优化

### 3.1 Content Visibility（PERF-7）

对于大型分类（元素数量多），使用 CSS `content-visibility` 优化离屏元素的渲染：

```css
/* 大分类时添加此类 */
.nav-grid-cv .nav-cell {
  content-visibility: auto;
  contain-intrinsic-size: 0 100px;  /* 预估尺寸 */
}
```

**优化效果**：
- 浏览器跳过离屏单元格的渲染/布局/绘制
- DOM 元素仍存在，dnd-kit 的注册、碰撞检测、键盘焦点不受影响
- 小网格不加此优化（切换成本无谓）

**判定阈值**（[navVirtualization.ts](../../frontend/src/utils/navVirtualization.ts)）：

```typescript
export function shouldUseContentVisibility(itemCount: number): boolean {
  return itemCount > 24;  // 超过 24 个元素时启用
}
```

### 3.2 React.memo 优化（PERF-2）

网格单元格使用多层 `React.memo` 优化：

```
NavGridCell (React.memo)
  └── NavCellContent (React.memo)
        ├── IconTile（图标）
        └── WidgetSlot（小组件）
```

**稳定回调**：

```typescript
// 将来自 Shell 的回调镜像进 ref，再用 useCallback([]) 暴露恒稳引用
const handlersRef = useRef({ onOpenIcon, onCtxTile, onExpandWidget });
handlersRef.current = { onOpenIcon, onCtxTile, onExpandWidget };

const handleOpenIcon = useCallback(
  (e: React.MouseEvent, x: IconView) =>
    handlersRef.current.onOpenIcon(e, x),
  []  // 空依赖 = 永不变
);
```

**优化效果**：拖拽/编辑某一项时，其余项的 props 全等不变，跳过重渲染。

### 3.3 分类切换滑入动画

切换分类时的滑入方向根据前后分类的索引位置自动判断：

```typescript
useEffect(() => {
  const prevIdx = groups.findIndex((g) => g.id === prevId);
  const nextIdx = groups.findIndex((g) => g.id === activeGroup);
  const dir = nextIdx > prevIdx ? 1 : -1;  // 1=向下, -1=向上
  setSlideDir(dir);
  setTimeout(() => setSlideDir(0), 380);  // 380ms 后清除动画类
}, [activeGroup, groups]);
```

## 4、响应式设计

### 4.1 网格列数

用户可自定义网格列数（通过 Tweaks 配置）：

```typescript
// 默认 6 列，可配置 4-12 列
<tweaks>
  gridCols = 6
</tweaks>
```

前端通过 CSS 变量应用：

```css
.nav-grid {
  grid-template-columns: repeat(var(--grid-cols, 6), 1fr);
}
```

### 4.2 图标区域宽度

用户可限制图标区域的最大宽度：

```typescript
// tweaks.iconAreaWidth: 0 或 undefined = 100%
// 其他值 = CSS 宽度值（如 "1200px"）
<div style={{ maxWidth: tweaks.iconAreaWidth || "100%" }}>
```

### 4.3 滚轮翻页

NavView 支持滚轮翻页（切换分类）：

```
当前分类已滚动到顶部/底部
         │
         ▼
    边界冷却 280ms（防止触控板惯性误触）
         │
         ▼
    累积 deltaY ≥ 阈值（默认 40）
         │
         ▼
    切换到上一个/下一个分类
    锁定 520ms（防止连续翻页）
```

**配置项**：
- `tweaks.wheelPage`：是否启用滚轮翻页
- `tweaks.wheelSensitivity`：翻页灵敏度（阈值，默认 40）

## 5、悬浮条小组件

### 5.1 特殊渲染

悬浮条小组件（如搜索框）不参与网格排序，独立渲染在网格顶部：

```tsx
<div className="nav-top-reserve has-bar">
  {floatingBars.map((fb) => (
    <div key={fb.id} className="floating-bar-slot">
      {WIDGET_REGISTRY[fb.widget].render(fb)}
    </div>
  ))}
</div>
```

**设计特点**：
- 横跨全部列（全宽）
- 不可拖动
- 始终固定在网格顶部
- 即使没有搜索小组件，顶部保留区也有最小高度（避免布局跳动）

### 5.2 注册为悬浮条

在 `WIDGET_REGISTRY` 中设置 `floatingBar: true`：

```typescript
search: {
  id: "search",
  name: "搜索",
  floatingBar: true,  // 标记为悬浮条模式
  render: (w) => <SearchWidget w={w} />,
}
```

## 6、空状态引导

### 6.1 两种空状态

| 场景 | 显示内容 | 操作按钮 |
|------|---------|---------|
| 无分类 | "还没有分类" + 引导文字 | "新建分类" |
| 分类内无内容 | "还没有图标" + 引导文字 | "添加图标" |

**可编辑判断**：仅在 `editable = true` 时显示操作按钮（非访客 + 分类可编辑）。

### 6.2 拖拽提示

首次使用时显示拖拽操作引导：

```
┌─────────────────────────────────────────────────────────────┐
│ ✨ 拖动图标可重新排序  │  📁 把图标深压到另一张上合并为文件夹  │
│ → 拖到左侧分类可移动到其他分类  │  ℹ️ 触摸屏长按图标再拖动    │
│                                                          ✕ │
└─────────────────────────────────────────────────────────────┘
```

**显示条件**：
- 可编辑 + 当前分类有内容 + 用户未关闭过

**持久化**：用户关闭后存储在 `localStorage`，不再显示。

## 7、前后端交互总结

| 操作 | 前端触发 | API 端点 | 说明 |
|------|----------|----------|------|
| 排序图标/小组件 | 拖拽松手 | `POST /api/groups/:id/reorder-items` | 分类内排序 |
| 移动到其他分类 | 拖到侧边栏 | `PATCH /api/icons/:id` | 修改 group_id |
| 合并为文件夹 | 深压到目标 | `POST /api/icons/:id/merge-into/:target` | 创建/追加文件夹 |
| 提取文件夹子项 | 拖出文件夹 | `POST /api/icons/:id/extract-item/:target` | 提取为独立图标 |
| 切换分类 | 点击侧边栏 | - | 前端本地切换 |
| 滚轮翻页 | 滚轮到底/顶 | - | 前端本地切换 |
