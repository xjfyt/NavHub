# 三、小组件

> **对应代码**：`frontend/src/widgets/`、`frontend/src/widgets/index.tsx`、`backend/src/handlers/widgets/`
> **维护提示**：修改小组件注册规范、数据刷新机制或新增小组件时同步更新本文档。

NavHub 的小组件（Widget）是嵌入导航网格中的轻量面板，承担"随手可查、随手可改"的交互职责。每个小组件可独立运行，支持懒加载、配置持久化、数据自动刷新等特性。

### 1.2 尺寸规格

小组件采用三档固定尺寸：

| 尺寸 ID | 名称 | 网格跨度 | 像素参考 | 适用场景 |
|---------|------|---------|---------|---------|
| `small` | 小 | 6×3 cells | ≈224×104px | 时钟、倒计时、番茄钟、年进度 |
| `medium` | 中 | 6×6 cells | ≈224×224px | 待办、便签、日历、热搜、计算器 |
| `large` | 大 | 12×5 cells | ≈464×184px | 天气、嵌入网页 |

**尺寸快照函数**（兼容旧数据）：

```typescript
// frontend/src/widgets/index.tsx
function snapWidgetSize(wSpan?: number, wRow?: number): WidgetSizeId {
  if (w === 6 && r === 3) return "small";
  if (w === 6 && r === 6) return "medium";
  if (w === 12 && r === 5) return "large";
  // 兜底：根据面积和宽高比推断
  const area = w * r;
  const aspect = w / r;
  if (aspect >= 1.7) return area >= 28 ? "large" : "small";
  return area >= 12 ? "medium" : "small";
}
```

## 2、注册规范

### 2.1 注册表结构

每个小组件必须在 `WIDGET_REGISTRY` 中注册：

```typescript
// frontend/src/widgets/index.tsx
interface WidgetTypeInfo {
  id: string;              // 唯一标识（英文小写，短横线分隔）
  name: string;            // 中文短名（≤ 10 字）
  description: string;     // 一句话介绍（≤ 40 字，用于搜索）
  icon?: string;           // Lucide 图标名
  defaultSize?: WidgetSizeId;  // 默认尺寸
  editable?: boolean;      // 是否支持配置编辑
  render: (w?: WidgetView) => React.ReactNode;          // 磁贴渲染
  renderDetail?: (w?: WidgetView) => React.ReactNode;   // 详情 Modal
  detailWidth?: string;    // 详情 Modal 宽度
  detailMaxHeight?: string; // 详情 Modal 最大高度
  floatingBar?: boolean;   // 是否为全宽悬浮条模式
}
```

## 3、小组件生命周期

### 3.1 懒加载

除搜索框外，所有小组件使用 `React.lazy` 延迟加载：

```typescript
const ClockWidget = lazy(() =>
  import("./ClockWidget").then((m) => ({ default: m.ClockWidget })),
);

// 包裹 Suspense 提供加载占位
function tile(node: React.ReactNode) {
  return (
    <Suspense fallback={<div className="widget-tile-fallback" aria-hidden />}>
      {node}
    </Suspense>
  );
}
```

**设计决策**：搜索框（`SearchWidget`）不懒加载，因为它在每个分类的悬浮条中渲染。

### 3.2 配置持久化

**Hook**：`useWidgetConfig`

```
用户修改配置 → 本地 useState 立即更新（乐观更新）
                    │
                    ▼ (600ms 防抖)
              PATCH /api/widgets/:id
                    │
                    ▼
              Database (widgets.config JSONB)
```

**规则**：
- 登录用户：写入数据库，跨设备/跨刷新保留
- 游客用户：纯内存，页面刷新丢失（不触发 401）

### 3.3 数据刷新

**Hook**：`useWidgetData`

| 小组件类型 | 刷新间隔 | 刷新方式 |
|-----------|---------|---------|
| 天气 | 30 分钟 | `useWidgetData(refreshMs)` |
| 热搜 | 5 分钟 | `useWidgetData(refreshMs)` |
| 一言 | 手动 | 不自动轮询 |
| 时钟/倒计时/年进度 | 1 秒/1 分钟 | 本地 `setInterval` |

**约束**：禁止 < 30 秒轮询（除时间类本地定时器）。

## 4、前端交互规范

### 4.1 统一骨架

```tsx
<div className="widget w-<kind>">
  <div className="widget-header">
    <span className="widget-title">{主标题}</span>
    <span className="muted mono" style={{ fontSize: 10 }}>{副状态}</span>
  </div>
  {主体内容}
</div>
```

### 4.2 三态处理

| 状态 | 副状态文字 | 主体区域 |
|------|-----------|---------|
| Loading | `LOADING` | 骨架/灰色占位 |
| Error | `ERROR` | 简短错误信息 |
| Empty | - | 引导文字（"点击齿轮添加…"） |

### 4.3 事件冒泡控制

小组件内的交互控件必须阻止事件冒泡，否则会触发磁贴的"展开详情"：

```tsx
// ✅ 正确：原生标签自动排除
<button onClick={handleClick}>点击</button>

// ✅ 正确：手动阻止冒泡
<div onClick={(e) => { e.stopPropagation(); handleClick(); }}>
  自定义按钮
</div>

// ✅ 正确：使用 data-nobubble 属性
<div data-nobubble onClick={handleClick}>自定义按钮</div>
```

### 4.4 可编辑 vs 可展开

| 能力 | 字段 | 触发方式 | UI 指示 |
|------|------|---------|--------|
| 配置编辑 | `editable: true` | 右键菜单 → 编辑 | 右键菜单显示齿轮 |
| 详情展开 | `renderDetail` | 点击磁贴空白区 | hover 显示 ⤢ |

两者独立：
- 时钟：可展开、不可编辑
- 番茄钟：可编辑 + 可展开
- 计算器：都不可（自身即完整交互）

## 5、后端代理

### 5.1 代理端点

| 端点 | 说明 | 缓存策略 |
|------|------|---------|
| `GET /api/widgets/weather` | 天气查询 | Redis 30 分钟 |
| `GET /api/widgets/hot` | 热搜榜单 | Redis 5 分钟 |
| `GET /api/widgets/music/search` | 音乐搜索 | Redis 10 分钟 |
| `GET /api/widgets/music/song/:id` | 歌曲详情 | Redis 30 分钟 |

### 5.2 外部 API 调用约定

- **优先走后端代理**：统一添加 Redis 缓存、隐藏 API Key
- **浏览器直连例外**：仅"官方支持 CORS 且无敏感 Key"的接口（如 `hitokoto.cn`）
- **缓存 TTL**：与刷新节奏挂钩（刷新 5 分钟 → 缓存 ≥ 5 分钟）

## 6、小组件 CRUD

### 6.1 创建小组件

**API 端点**：`POST /api/widgets`

**请求体**：

```json
{
  "groupId": "uuid",
  "widget": "weather",
  "wSpan": 6,
  "wRow": 6,
  "config": { "city": "北京" }
}
```

**后端逻辑**（[widgets/crud.rs](../../backend/src/handlers/widgets/crud.rs)）：

1. 验证分类写权限
2. 插入数据库，`sort_order` 自动取最大值 + 1
3. 返回 `WidgetView`

### 6.2 更新小组件

**API 端点**：`PATCH /api/widgets/:id`

支持更新：`group_id`（跨分类移动）、`w_span`/`w_row`（调整大小）、`config`（配置变更）、`sort_order`。

### 6.3 删除小组件

**API 端点**：`DELETE /api/widgets/:id`

## 7、新增小组件开发指南

### 7.1 开发清单

- [ ] 使用 `.widget` + `.widget-header` 骨架
- [ ] 至少一种磁贴内交互，或者有 `renderDetail`
- [ ] 用 `useWidgetConfig` 持久化配置（如无配置可跳过）
- [ ] 使用 `useWidgetData`（远端数据）或本地 `setInterval`（时间类）
- [ ] 处理 Loading / Error / Empty 三态
- [ ] 交互控件 `e.stopPropagation()`
- [ ] 登录用户与游客都不报错
- [ ] 注册到 `WIDGET_REGISTRY`，并填 `DEMO_CONFIG`
- [ ] `name` / `description` 包含搜索关键词
- [ ] `npx tsc --noEmit` & `npx vite build` 通过

### 7.2 最小示例

```tsx
// frontend/src/widgets/MyWidget.tsx
import { useWidgetConfig } from "../hooks/useWidgetConfig";
import { useWidgetData } from "../hooks/useWidgetData";

export function MyWidget({ w }: { w?: WidgetView }) {
  const [config, setConfig] = useWidgetConfig(w);
  const { data, loading, error } = useWidgetData("/api/widgets/my-data", {
    refreshMs: 5 * 60 * 1000, // 5 分钟
  });

  return (
    <div className="widget w-my">
      <div className="widget-header">
        <span className="widget-title">我的小组件</span>
        <span className="muted mono" style={{ fontSize: 10 }}>
          {loading ? "LOADING" : error ? "ERROR" : "OK"}
        </span>
      </div>
      {loading ? <div className="skeleton" /> :
       error ? <div>{error.message}</div> :
       <div>{/* 正常内容 */}</div>
      }
    </div>
  );
}
```

## 8、前后端交互总结

| 操作 | 触发方式 | API 端点 |
|------|----------|----------|
| 创建小组件 | 目录弹窗选择 | `POST /api/widgets` |
| 更新配置 | 齿轮弹窗编辑 | `PATCH /api/widgets/:id` |
| 调整大小 | 右键菜单 | `PATCH /api/widgets/:id` |
| 删除小组件 | 右键菜单 | `DELETE /api/widgets/:id` |
| 排序 | 拖拽松手 | `POST /api/groups/:id/reorder-items` |
| 获取天气 | 前端定时拉取 | `GET /api/widgets/weather` |
| 获取热搜 | 前端定时拉取 | `GET /api/widgets/hot` |
