# 二、技术栈详解

> **对应代码**：`backend/Cargo.toml`、`frontend/package.json`、`Dockerfile`
> **维护提示**：技术栈版本升级或新增依赖时同步更新本文档。

## 1、后端技术栈

### 1.1 Rust + Axum

| 组件 | 版本 | 用途 |
|------|------|------|
| Rust | 1.95.0 | 主要编程语言，由 `rust-toolchain.toml` 锁定版本 |
| Axum | 0.7 | Web 框架，基于 tokio 生态的异步 HTTP 服务 |

**为什么选择 Rust + Axum：**

- **高性能**：零成本抽象、无 GC 停顿，适合自托管场景下有限资源的高效利用
- **内存安全**：所有权系统在编译期消除大部分内存错误
- **异步生态**：与 tokio 深度集成，天然支持高并发连接
- **Axum 优势**：基于 tower 中间件体系，路由提取器模式开发体验优秀

**关键依赖：**

```toml
[dependencies]
axum = "0.7"
tokio = { version = "1", features = ["full"] }
tower = "0.4"
tower-http = { version = "0.5", features = ["cors", "trace"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

### 1.2 SQLx + PostgreSQL

| 组件 | 版本 | 用途 |
|------|------|------|
| SQLx | 0.8 | 异步数据库访问层，编译期校验 SQL |
| PostgreSQL | 15 | 主数据库，存储所有业务数据 |

**SQLx 特性：**

- **编译期 SQL 检查**：通过 `sqlx::query!` 宏在编译时校验 SQL 语法和字段类型
- **异步连接池**：内置连接池管理，支持配置最大连接数
- **自动迁移**：启动时自动执行 `backend/migrations/` 目录下未执行的迁移脚本
- **事务支持**：支持手动事务管理，保证数据一致性

**数据库配置示例（config.toml）：**

```toml
[database]
host = "127.0.0.1"
port = 5432
user = "navhub"
password = "your_password"
database = "navhub"
max_connections = 10
```

### 1.3 Redis

| 组件 | 版本 | 用途 |
|------|------|------|
| Redis | 7 | 缓存、会话存储、运行时状态管理 |

**Redis 用途：**

- **会话存储**：`session:{uuid}` → `{user_id, role}`，TTL 7 天
- **缓存层**：热点数据缓存，减少数据库查询
- **限流计数**：登录接口限流（令牌桶算法）
- **运行时配置**：超级管理员在后台修改的 SSO 配置等

### 1.4 认证模块

| 组件 | 用途 |
|------|------|
| Casdoor | OIDC 单点登录服务端 |
| Argon2 | 密码哈希算法（超级管理员本地密码） |

**认证流程：**

```
Browser ──(1) /auth/login──> Rust ──(2) 302──> Casdoor
       <──(5) Set-Cookie── <──(4) token/userinfo── ↑
       ──(3) /auth/callback?code=...────────────── ↓
Browser ──(6) /api/*──> Rust ──> PG / Redis
```

- **普通用户**：仅可通过 OIDC 单点登录（Casdoor / Authentik 等兼容服务）
- **超级管理员**：支持 OIDC + 账号密码双通道登录
- **Session Cookie**：`nh_sid`，属性 `HttpOnly; Secure; SameSite=Lax`

## 2、前端技术栈

### 2.1 React 18 + TypeScript 5

| 组件 | 版本 | 用途 |
|------|------|------|
| React | 18 | UI 框架，使用函数组件 + Hooks |
| TypeScript | 5 | 类型安全，减少运行时错误 |

**前端架构特点：**

- **单页应用（SPA）**：所有页面在一个 React 应用内，通过路由切换
- **类型驱动开发**：所有 API 接口都有对应的 TypeScript 类型定义
- **组件化设计**：核心 UI 拆分为可复用组件（Workspace、GroupSidebar、IconGrid 等）

### 2.2 Vite 5

| 组件 | 版本 | 用途 |
|------|------|------|
| Vite | 5 | 构建工具，开发服务器 + 生产打包 |

**Vite 优势：**

- **极速 HMR**：基于 ESM 的模块热替换，开发时即时刷新
- **按需编译**：开发环境只编译访问的模块，启动速度快
- **优化打包**：生产环境基于 Rollup 打包，支持代码分割和 Tree Shaking

**开发模式下：**

- 前端开发服务器：`http://127.0.0.1:5173`（Vite HMR）
- 后端 API 代理：Vite 将 `/api/*` 请求代理到 `http://127.0.0.1:8088`

### 2.3 核心前端依赖

| 依赖 | 用途 |
|------|------|
| @dnd-kit | 拖拽功能（图标排序、文件夹合并、分类移动） |
| Tailwind CSS | 原子化 CSS 框架 |
| React Router | 客户端路由 |

## 3、存储架构

### 3.1 三层存储模型

| 层 | 存储引擎 | 数据类型 |
|----|----------|----------|
| 主数据 | PostgreSQL | 用户、图标、分组、偏好设置、审计日志 |
| 缓存/会话 | Redis | Session、限流计数、运行时配置 |
| 文件存储 | S3 兼容对象存储 | 用户上传的图标图片、壁纸资源 |

### 3.2 S3 对象存储

NavHub 支持任何 S3 兼容的对象存储服务（AWS S3、MinIO、Cloudflare R2 等）。

**配置示例（config.toml）：**

```toml
[storage.s3]
endpoint = "https://minio.example.com"
region = "us-east-1"
bucket = "navhub"
access_key = "your_access_key"
secret_key = "your_secret_key"
path_style = true
key_prefix = "assets"
presign_ttl_secs = 86400
```

**S3 存储特性：**

- **内容哈希文件名**：上传文件使用 SHA256 内容哈希命名，避免冲突
- **预签名 URL**：通过 presigned URL 提供文件访问，24 小时有效
- **路径风格**：支持 path-style 访问（如 MinIO 默认配置）
