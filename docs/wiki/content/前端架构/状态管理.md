# 二、状态管理

> **对应代码**：`frontend/src/hooks/useWorkspace.tsx`、`frontend/src/App.tsx`、`frontend/src/utils/connectivity.ts`、`frontend/src/utils/undoQueue.ts`
> **维护提示**：新增/修改全局状态、Context 结构或数据流时同步更新本文档。

## 1、状态管理概述

NavHub 采用 **React 内置状态管理** 方案，不依赖 Redux、Zustand 等外部库。核心策略：

| 层级 | 技术 | 职责 |
|------|------|------|
| 全局状态 | `WorkspaceDataContext` | 工作区数据（groups、icons、widgets、preferences） |
| 操作函数 | `WorkspaceActionsContext` | 状态变更操作（CRUD、排序、拖拽） |
| 页面状态 | `useState` / `useReducer` | 当前组件的局部状态 |
| 连接状态 | `useReducer` | 在线/离线、后端可达性 |
| 缓存层 | `localStorage`（SWR） | 持久化工作区快照 |

## 2、Context 分离设计

### （1）双 Context 架构

为避免不必要的重渲染，`useWorkspace` 将数据与操作分离为两个 Context：

```typescript
// hooks/useWorkspace.tsx
interface WorkspaceDataContextProps {
  me: Me | null;
  isGuest: boolean;
  workspace: Workspace;
  activeGroup: string;
}

interface WorkspaceActionsContextProps {
  setActiveGroup: (id: string) => void;
  updateTweaks: (t: Partial<Tweaks>) => void;
  reorderGroup: (oldId: string, newId: string) => void;
  updateIcon: (id: string, patch: Partial<IconView>) => Promise<void>;
  deleteIcon: (id: string) => Promise<void>;
  addWidget: (groupId: string, kind: string, size?: WidgetSizeId) => Promise<void>;
  // ... 更多操作
}
```

**设计收益**：

- 只消费 `ActionsContext` 的组件（如模态框）不会因工作区数据变化而重渲染
- 操作函数引用稳定（使用 `latestRef` 模式），跨渲染不重建

### （2）useWorkspace Hook

对外暴露合并后的统一接口：

```typescript
export function useWorkspace(): WorkspaceContextProps {
  const data = useContext(WorkspaceDataContext);
  const actions = useContext(WorkspaceActionsContext);
  if (!data || !actions) {
    throw new Error("useWorkspace must be used within <WorkspaceProvider>");
  }
  return { ...data, ...actions };
}
```

## 3、数据流模式

### （1）乐观更新（Optimistic Update）

用户操作后立即更新 UI，同时异步发送 API 请求：

```typescript
// hooks/useWorkspace.tsx - updateIcon 示例
const updateIcon = useCallback(async (id: string, patch: Partial<IconView>) => {
  // 1. 乐观更新本地状态
  setWorkspace(s => ({
    ...s,
    icons: arrUpdate(s.icons, id, patch),
  }));
  
  // 2. 异步同步到后端
  try {
    await api.updateIcon(id, patch);
  } catch (e) {
    // 3. 失败时回滚（通过 refreshWorkspace）
    console.error("updateIcon failed", e);
    toast.error("保存失败");
    onReload?.();
  }
}, []);
```

### （2）延迟删除 + 撤销

危险操作（如删除图标/组件）采用延迟落库策略：

```typescript
// hooks/useWorkspace.tsx - deleteIcon 示例
const deleteIcon = useCallback(async (id: string) => {
  const icon = latestRef.current.workspace.icons.find(x => x.id === id);
  if (!icon) return;
  
  // 1. 乐观移除
  setWorkspace(s => ({ ...s, icons: arrRemove(s.icons, id) }));
  
  // 2. 加入撤销队列（5 秒内可撤销）
  undoQueueRef.current.enqueue({
    kind: "icon",
    id,
    undo: () => {
      // 撤销：恢复到原位
      setWorkspace(s => ({ ...s, icons: [...s.icons, icon].sort((a, b) => a.sortOrder - b.sortOrder) }));
    },
    commit: async () => {
      // 落库：真正删除
      await api.deleteIcon(id);
    },
  });
}, []);
```

**撤销队列**（`undoQueue.ts`）：

```typescript
// utils/undoQueue.ts
export function createUndoQueue(opts: { delayMs: number }) {
  const pending = new Map<string, PendingTask>();
  
  return {
    enqueue(task: PendingTask) {
      pending.set(task.id, task);
      setTimeout(() => {
        task.commit();
        pending.delete(task.id);
      }, opts.delayMs);
    },
    undo(id: string) {
      const task = pending.get(id);
      if (task) {
        task.undo();
        pending.delete(id);
      }
    },
    flushAll() {
      // 卸载时立即落库所有未决删除
      for (const task of pending.values()) {
        task.commit();
      }
      pending.clear();
    },
  };
}
```

### （3）latestRef 模式

避免操作函数因状态变化而重建引用：

```typescript
// hooks/useWorkspace.tsx
const latestRef = useRef({ workspace, isGuest, onReload });
latestRef.current = { workspace, isGuest, onReload };

// 操作函数从 ref 读取最新状态，依赖数组为空
const canEditGroup = useCallback((id: string) => {
  const { isGuest: guest, workspace: ws } = latestRef.current;
  if (guest) return false;
  const g = ws.groups.find(x => x.id === id);
  return g ? !g.readOnly : false;
}, []); // 空依赖 → 引用永不变化
```

## 4、SWR 缓存机制

### （1）缓存读写

启动时优先渲染缓存数据，同时在后台重新验证：

```typescript
// App.tsx
const SWR_KEY = "navhub_swr_v1";

function readSwr(): SwrPayload | null {
  const raw = window.localStorage.getItem(SWR_KEY);
  if (!raw) return null;
  const parsed = JSON.parse(raw);
  // 结构校验防止损坏数据崩溃
  if (parsed?.workspace?.groups && parsed?.status) {
    return parsed as SwrPayload;
  }
  return null;
}

function writeSwr(payload: SwrPayload) {
  window.localStorage.setItem(SWR_KEY, JSON.stringify(payload));
}
```

### （2）启动流程

```
┌─────────────────────────────────────────────────────────────┐
│                      App 启动                               │
├─────────────────────────────────────────────────────────────┤
│ 1. readSwr() → 有缓存？                                     │
│    ├─ 是 → 立即渲染缓存数据 + revalidating=true              │
│    └─ 否 → 渲染空骨架（emptyWorkspace）                      │
│                                                             │
│ 2. 并行请求：                                                │
│    ├─ api.status()    → 认证状态                             │
│    ├─ api.workspace() → 工作区数据                           │
│    └─ api.me()        → 用户信息（仅已登录时）                │
│                                                             │
│ 3. 请求完成 → setState + writeSwr + setRevalidating(false)   │
│    └─ 失败 → 保留当前 UI + dispatchConn("backend_error")     │
└─────────────────────────────────────────────────────────────┘
```

## 5、连接状态管理

使用 `useReducer` 管理在线/离线 + 后端可达性：

```typescript
// App.tsx
const [conn, dispatchConn] = useReducer(connectivityReducer, navigator.onLine, initialConnectivity);

// utils/connectivity.ts
type ConnectivityState = {
  online: boolean;
  backend: "ok" | "unreachable" | "unknown";
  banner: BannerState | null;
};

function connectivityReducer(state: ConnectivityState, action: ConnectivityAction): ConnectivityState {
  switch (action.type) {
    case "online":
      return { ...state, online: true };
    case "offline":
      return { ...state, online: false, banner: { kind: "offline", message: "网络已断开" } };
    case "backend_ok":
      return { ...state, backend: "ok", banner: null };
    case "backend_error":
      return { ...state, backend: "unreachable", banner: { kind: "backend", message: "后端不可达" } };
  }
}
```

### （2）缓存策略

| 场景 | 行为 |
|------|------|
| 首次加载 | 发起网络请求，写入共享缓存 |
| 重复挂载（同 cacheKey） | 直接读缓存，零网络 |
| 手动刷新 | 失效缓存 + 重新请求 |
| 定时刷新 | `setInterval` + `forceFresh=true` |
| 组件卸载 | `AbortController.abort()` 中止请求 |

## 7、Guest 模式状态

访客用户的状态持久化策略：

```typescript
// hooks/useWorkspace.tsx
const updateTweaks = useCallback(async (t: Partial<Tweaks>) => {
  setWorkspace(s => ({
    ...s,
    preferences: { ...s.preferences, tweaks: { ...s.preferences.tweaks, ...t } },
  }));
  
  if (latestRef.current.isGuest) {
    // 访客：写入 localStorage
    window.localStorage.setItem("navhub_guest_tweaks", JSON.stringify(nextTweaks));
    return;
  }
  
  // 已登录：同步到后端
  await api.patchPrefs({ tweaks: nextTweaks });
}, []);
```

// App.tsx - boot()
if (!me) {
