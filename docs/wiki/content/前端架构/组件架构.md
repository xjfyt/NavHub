# 一、组件架构

> **对应代码**：`frontend/src/App.tsx`、`frontend/src/WorkspaceScreen.tsx`、`frontend/src/components/Shell.tsx`、`frontend/src/components/Sidebar.tsx`、`frontend/src/components/NavView.tsx`
> **维护提示**：新增/移除核心组件、修改组件层次结构或数据流时同步更新本文档。

## 1、组件层次总览

NavHub 前端采用单页应用（SPA）架构，组件树自顶向下分为三层：**入口层** → **外壳层** → **功能层**。

```
App (入口层)
├── WorkspaceScreen (工作区入口)
│   └── WorkspaceProvider (Context 容器)
│       └── Shell (外壳层)
│           ├── Background (背景/壁纸)
│           ├── Sidebar (侧边栏导航)
│           ├── NavView (主内容区)
│           │   ├── IconTile (图标磁贴)
│           │   └── WidgetTile (组件磁贴)
│           ├── SearchBar (搜索栏)
│           └── 各类 Modal/Overlay (功能层)
├── LoginScreen (登录页)
├── ChangePasswordScreen (强制改密页)
└── Toaster (全局通知)
```

## 2、入口层组件

### （1）App 组件

`App` 是应用根组件，负责：

- **启动引导**（`boot`）：并行请求 `/auth/status`、`/api/workspace`、`/api/me`
- **SWR 缓存**：将工作区数据写入 `localStorage`，下次访问时立即渲染缓存数据
- **连接状态管理**：监听 `online/offline` 事件，渲染全局横幅
- **路由分发**：根据启动阶段（`BootState`）决定渲染哪个页面

```typescript
// App.tsx 核心状态
type BootState =
  | { stage: "error"; message: string }
  | { stage: "must_change_password" }
  | { stage: "ready"; status: AuthStatus | null; me: Me | null; workspace: Workspace };
```

**关键设计**：

| 设计点 | 说明 |
|--------|------|
| 冷启动优化 | 首次访问时渲染空骨架（`emptyWorkspace()`），不阻塞首屏 |
| SWR 读写 | `readSwr()` 读取缓存时做结构校验，防止损坏数据崩溃 |
| 连接状态 | `useReducer` 管理 `online/backend_ok/backend_error/offline` 四态 |

### （2）WorkspaceScreen 组件

薄包装层，将 `me`、`workspace` 注入 `WorkspaceProvider`，再渲染 `Shell`。

```typescript
// WorkspaceScreen.tsx
export function WorkspaceScreen(props: {
  me: Me | null;
  workspace: Workspace;
  onReload: () => void;
  onRequestLogin: () => void;
  onLogout: () => void | Promise<void>;
}) {
  return (
    <WorkspaceProvider initialMe={me} initialWorkspace={workspace} onReload={onReload}>
      <Shell onLogout={onLogout} onRequestLogin={onRequestLogin} />
    </WorkspaceProvider>
  );
}
```

## 3、外壳层组件

### （1）Shell 组件

`Shell` 是应用的"骨架"，承担以下职责：

- **布局管理**：侧边栏 + 主内容区的双栏布局
- **拖拽协调**：单一 `<DndContext>` 同时覆盖侧边栏与网格，统一处理分类内排序、文件夹合并、跨分类移动
- **模态框编排**：管理十余个模态框/覆盖层的开关状态
- **右键菜单**：统一的上下文菜单系统

```typescript
// Shell.tsx 关键状态
const [tweaksOpen, setTweaksOpen] = useState(false);
const [adminOpen, setAdminOpen] = useState(false);
const [catalogOpen, setCatalogOpen] = useState(false);
const [addCatOpen, setAddCatOpen] = useState<boolean | GroupView>(false);
const [addIconOpen, setAddIconOpen] = useState<boolean | IconView>(false);
const [ctxMenu, setCtxMenu] = useState<CtxMenuState | null>(null);
// ... 更多模态框状态
```

**懒加载策略**：所有非首屏组件均使用 `React.lazy` + `Suspense` 按需加载：

```typescript
const TweaksPanel = lazy(() => import("./TweaksPanel").then(m => ({ default: m.TweaksPanel })));
const AdminShell = lazy(() => import("./admin").then(m => ({ default: m.AdminShell })));
const AddIconModal = lazy(() => import("./AddIconModal").then(m => ({ default: m.AddIconModal })));
// ...
```

### （2）Sidebar 组件

侧边栏负责：

- 分类列表渲染与切换
- 用户头像与菜单入口
- 拖拽排序分类（`useDroppable`）
- 移动端抽屉模式

### （3）NavView 组件

主内容区，展示当前分类下的图标网格和组件网格：

- 响应式网格布局（CSS Grid）
- 拖拽排序（`@dnd-kit/sortable`）
- 空状态引导

## 4、功能层组件

### （1）图标组件

| 组件 | 职责 |
|------|------|
| `Icon` | 通用图标渲染（字母/图片/文件夹） |
| `IconTile` | 图标磁贴（含标题、副标题、点击事件） |
| `FolderOverlay` | 文件夹展开覆盖层 |

### （2）组件（Widget）系统

Widget 注册表（`WIDGET_REGISTRY`）统一管理所有组件类型：

```typescript
// widgets/index.tsx
export const WIDGET_REGISTRY: Record<string, WidgetDef> = {
  clock:      { Component: ClockWidget, Detail: ClockDetail, ... },
  weather:    { Component: WeatherWidget, Detail: WeatherDetail, ... },
  todo:       { Component: TodoWidget, Detail: TodoDetail, ... },
  countdown:  { Component: CountdownWidget, Detail: CountdownDetail, ... },
  // ... 更多组件
};
```

每个 Widget 遵循统一接口：

```typescript
interface WidgetDef {
  Component: React.LazyExoticComponent<React.ComponentType<WidgetProps>>;
  Detail?: React.LazyExoticComponent<React.ComponentType<WidgetProps>>;
  editable?: boolean;
  defaultSize?: WidgetSizeId;
  sizes?: WidgetSizeId[];
  nameKey: string;  // i18n key
  icon: string;     // lucide icon name
}
```

### （3）模态框组件

所有模态框采用 `ModalSuspense` 包装，加载期间返回 `null` 避免闪烁：

```typescript
const ModalSuspense = ({ children }: { children: React.ReactNode }) => (
  <Suspense fallback={null}>{children}</Suspense>
);
```

## 5、组件通信模式

### （1）Props 下传 + 回调上抛

标准父子通信模式，如 `Shell` → `NavView`：

```typescript
<NavView
  activeGroup={activeGroup}
  icons={workspace.icons}
  onOpenIcon={(_e, ic) => openIcon(ic)}
  onCtxTile={tileCtx}
  // ...
/>
```

### （2）Context 共享

通过 `useWorkspace()` hook 共享工作区状态与操作函数，详见 [状态管理.md](./状态管理.md)。

### （3）右键菜单系统

`Shell.menus.tsx` 中定义菜单构建函数，接收 `ShellMenuCtx` 上下文对象：

```typescript
const menuCtx: ShellMenuCtx = {
  isGuest, activeGroup, groups, canEditGroup,
  openCtx, openIcon, updateIcon, deleteIcon, // ...
};
const tileCtx = (e, item) => buildTileCtx(menuCtx, e, item);
```

## 6、代码分割策略

| 分割层级 | 策略 | 示例 |
|----------|------|------|
| 页面级 | `App` 按 `BootState` 条件渲染 | `LoginScreen`、`ChangePasswordShell` |
| 功能级 | `React.lazy` 按需加载 | `AdminShell`、`TweaksPanel` |
| 组件级 | Widget 内部 `lazy` | `ClockWidget`、`WeatherWidget` |
| 模态框级 | `ModalSuspense` 包装 | 所有 Modal/Overlay |

**性能收益**：首屏仅加载 `Shell` + `Sidebar` + `NavView` 核心代码，管理后台、设置面板等重模块在用户触发时才加载。
