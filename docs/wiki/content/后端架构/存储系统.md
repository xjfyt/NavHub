# 五、存储系统

> **对应代码**：`backend/src/storage.rs`、`backend/src/handlers/upload.rs`
> **维护提示**：修改 S3 集成、上传逻辑时同步更新本文档。

## 1、架构概述

NavHub 使用 S3 兼容的对象存储服务管理用户上传的文件（图标、壁纸、头像等）。
支持任何 S3 兼容的存储后端，包括 MinIO、AWS S3、Cloudflare R2 等。

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  浏览器   │ ──→ │  /api/upload  │ ──→ │  S3 存储     │
│          │ ←── │  (Handler)    │ ←── │  (MinIO等)   │
└──────────┘     └──────────────┘     └──────────────┘
```

## 2、配置

```toml
[storage.s3]
endpoint = "http://localhost:9000"  # S3 端点
region = "us-east-1"               # 区域
bucket = "navhub"                   # 桶名
access_key = "..."                  # 访问密钥
secret_key = "..."                  # 秘密密钥
path_style = true                    # 路径样式（MinIO 需要）
key_prefix = "assets"               # 对象键前缀
presign_ttl_secs = 3600              # 预签名 URL 有效期
```

## 3、核心功能

### （1）对象上传（put_bytes）

上传文件到 S3，自动生成对象键：

```rust
pub async fn put_bytes(
    &self,
    public_name: &str,       // 文件名（不含前缀）
    content_type: Option<&str>, // MIME 类型
    data: Bytes,             // 文件内容
) -> AppResult<()>
```

### （2）预签名 URL（presign_get_url）

生成临时访问 URL，用于前端直接从 S3 下载文件，无需经过后端代理。

- 有效期由 `presign_ttl_secs` 配置，默认 3600 秒
- 有效期被夹取到 [60s, 86400s] 区间，防止配置错误

### （3）对象删除

- **单个删除**：`delete_object(public_name)`，幂等操作
- **批量删除**：`delete_objects(public_names)`，每批最多 1000 个 key
- **S3 DeleteObjects** 对不存在的 key 返回成功，因此重复清理是安全的

### （4）健康检查

使用 HeadBucket 探测 S3 可达性，供 /readyz 端点可选启用。

## 4、对象键管理

### （1）键格式

对象键由 key_prefix + public_name 组成：

- 配置 `key_prefix = "assets"`
- 上传 `public_name = "icons/abc123.png"`
- 最终键：`assets/icons/abc123.png`

### （2）键安全（sanitize_public_name）

public_name 经过严格校验，防止路径穿越攻击：

- 去除首尾空白和斜杠
- 拒绝空字符串、包含 .. 或 \ 的路径
- 拒绝以 / 开头的绝对路径

### （3）存储值还原（key_from_stored_value）

数据库中存储的 URL 可能是相对路径或完整 URL，还原为 S3 对象键时：

- 相对路径：`/uploads/icons/abc.png` → `icons/abc.png`
- 完整 URL：`https://cdn.example.com/uploads/icons/abc.png` → `icons/abc.png`
- 去除查询参数和片段
- 非 uploads 路径或路径穿越返回 None（安全拒绝）

## 5、集成场景

| 场景 | 说明 |
|------|------|
| 用户头像 | 上传到 S3，URL 存入 users.avatar_url |
| 图标图标 | 上传到 S3，URL 存入 icons.image_url |
| 壁纸文件 | 由壁纸源抓取后上传，URL 存入 remote_wallpapers.storage_key |
| 图标库资源 | 由图标源抓取后上传，URL 存入 library_icons.url |

## 6、清理策略

当数据库记录被删除时，对应的 S3 对象也需要清理：

- **用户删除**：清理头像 S3 对象
- **图标删除**：清理图标 S3 对象
- **壁纸过期**：后台任务分批清理过期壁纸及其缩略图
- **孤立图标 GC**：定期清理未被引用的临时上传图标
