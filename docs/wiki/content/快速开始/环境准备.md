# 四、环境准备

> **对应代码**：`backend/rust-toolchain.toml`、`frontend/package.json`、`config.example.toml`
> **维护提示**：依赖版本变更或新增前置工具时同步更新本文档。

## 1、硬件要求

### 1.1 最低配置

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| CPU | 1 核 | 2 核 |
| 内存 | 512 MB | 1 GB |
| 磁盘 | 1 GB | 5 GB |
| 网络 | 能访问 PostgreSQL 和 Redis | 低延迟内网 |

### 1.2 操作系统支持

- **Linux**：Ubuntu 20.04+、Debian 11+、CentOS 8+（推荐）
- **macOS**：12+（Monterey 及以上）
- **Windows**：10/11（WSL2 推荐）

## 2、软件依赖

### 2.1 必需工具

| 工具 | 版本要求 | 安装说明 |
|------|----------|----------|
| Rust | >= 1.95.0 | 通过 `rustup` 安装 |
| Node.js | >= 20 | 推荐使用 nvm 管理版本 |
| pnpm 或 npm | 任意 | pnpm 更快，npm 随 Node.js 附带 |
| PostgreSQL | >= 15 | 主数据库 |
| Redis | >= 7 | 缓存和会话存储 |

### 2.2 可选工具

| 工具 | 用途 | 安装说明 |
|------|------|----------|
| Docker | 容器化部署 | [docker.com](https://docs.docker.com/get-docker/) |
| sqlx-cli | 数据库迁移管理 | `cargo install sqlx-cli` |
| cargo-watch | 后端热重载 | `cargo install cargo-watch` |
| MinIO | 本地 S3 存储 | [min.io](https://min.io/download) |

## 3、安装 Rust

### 3.1 Linux / macOS

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env
```

### 3.2 Windows

下载并运行 [rustup-init.exe](https://rustup.rs)，或使用 winget：

```powershell
winget install Rustlang.Rustup
```

### 3.3 验证安装

```bash
rustc --version
# 应输出类似：rustc 1.95.0 (xxxxxx 2025-xx-xx)

cargo --version
# 应输出类似：cargo 1.95.0 (xxxxxx 2025-xx-xx)
```

**注意**：项目根目录的 `backend/rust-toolchain.toml` 会自动锁定 Rust 版本为 1.95.0，无需手动指定。

## 4、安装 Node.js

### 4.1 使用 nvm（推荐）

```bash
# Linux / macOS
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
```

### 4.2 Windows（使用 nvm-windows）

```powershell
# 使用 winget
winget install CoreyButler.NVMforWindows

# 重启终端后
nvm install 20
nvm use 20
```

### 4.3 验证安装

```bash
node --version
# 应输出：v20.x.x

npm --version
# 应输出：10.x.x
```

## 5、安装 PostgreSQL

### 5.1 Docker 方式（推荐开发环境）

```bash
docker run -d \
  --name navhub-pg \
  -e POSTGRES_USER=navhub \
  -e POSTGRES_PASSWORD=navhub_dev \
  -e POSTGRES_DB=navhub \
  -p 5432:5432 \
  postgres:15-alpine
```

### 5.2 系统安装

**Ubuntu / Debian：**

```bash
sudo apt update
sudo apt install postgresql-15
sudo systemctl start postgresql
```

**macOS（Homebrew）：**

```bash
brew install postgresql@15
brew services start postgresql@15
```

### 5.3 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 在 psql 中执行
CREATE USER navhub WITH PASSWORD 'your_password';
CREATE DATABASE navhub OWNER navhub;
\q
```

## 6、安装 Redis

### 6.1 Docker 方式（推荐开发环境）

```bash
docker run -d \
  --name navhub-redis \
  -p 6379:6379 \
  redis:7-alpine
```

### 6.2 系统安装

**Ubuntu / Debian：**

```bash
sudo apt update
sudo apt install redis-server
sudo systemctl start redis-server
```

**macOS（Homebrew）：**

```bash
brew install redis
brew services start redis
```

### 6.3 验证连接

```bash
redis-cli ping
# 应输出：PONG
```

## 7、安装 S3 存储（可选）

如果需要使用文件上传功能（图标图片、壁纸等），需要配置 S3 兼容的对象存储。

### 7.1 MinIO（本地开发推荐）

```bash
docker run -d \
  --name navhub-minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio server /data --console-address ":9001"
```

访问 `http://localhost:9001` 创建 bucket（如 `navhub`）和 access key。

### 7.2 其他 S3 兼容服务

- **AWS S3**：生产环境推荐
- **Cloudflare R3**：免出站流量费用
- **阿里云 OSS**：国内部署推荐

## 8、配置文件

### 8.1 复制示例配置

```bash
cp config.example.toml config.toml
```

### 8.2 最小化开发配置

编辑 `config.toml`，只需修改数据库和 Redis 连接信息：

```toml
[server]
host = "0.0.0.0"
port = 8088
public_url = "http://localhost:8088"

[database]
host = "127.0.0.1"
port = 5432
user = "navhub"
password = "your_password"
database = "navhub"
max_connections = 10

[redis]
host = "127.0.0.1"
port = 6379
password = ""
db = 0

[superadmin]
username = "superadmin"
password = "superadmin"
email = "admin@example.com"
```

### 8.3 环境变量覆盖

所有配置项均可通过环境变量覆盖，格式为 `NAVHUB__<SECTION>__<KEY>`：

```bash
# 示例
export NAVHUB__DATABASE__HOST=192.168.1.100
export NAVHUB__DATABASE__PASSWORD=secret
export NAVHUB__SERVER__PORT=9000
```

## 9、环境检查清单

完成上述安装后，运行以下命令确认环境就绪：

```bash
# Rust 工具链
rustc --version && cargo --version

# Node.js 和包管理器
node --version && npm --version

# PostgreSQL 连接测试
psql -h 127.0.0.1 -p 5432 -U navhub -d navhub -c "SELECT 1;"

# Redis 连接测试
redis-cli -h 127.0.0.1 ping
```

全部通过后，即可进入 [本地开发](./本地开发.md) 章节。
