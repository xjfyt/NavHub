# 六、生产部署

> **对应代码**：`Dockerfile`、`docker-compose.yaml`、`config.example.toml`
> **维护提示**：部署流程、Docker 配置或环境变量变更时同步更新本文档。

## 1、部署方式概览

NavHub 支持多种部署方式，推荐根据实际需求选择：

| 方式 | 适用场景 | 复杂度 |
|------|----------|--------|
| docker-compose | 一键启动全部服务（含 PG、Redis） | 低 |
| 纯 Docker | 已有外部 PG、Redis，仅部署应用 | 中 |
| 手动构建 | 无 Docker 环境，需要从源码构建 | 高 |

## 2、docker-compose 部署（推荐）

### 2.1 准备工作

```bash
mkdir -p /opt/navhub && cd /opt/navhub
wget https://raw.githubusercontent.com/xjfyt/navhub/main/config.example.toml -O config.toml
wget https://raw.githubusercontent.com/xjfyt/navhub/main/docker-compose.yaml
```

### 2.2 修改配置

编辑 `config.toml`，将数据库和 Redis 的 host 改为 Docker 服务名：

```toml
[database]
host = "postgres"
port = 5432
user = "navhub"
password = "your_secure_password"
database = "navhub"

[redis]
host = "redis"
port = 6379
password = ""
db = 0

[server]
host = "0.0.0.0"
port = 8088
public_url = "https://navhub.example.com"
```

### 2.3 启动服务

```bash
docker compose up -d
docker compose logs -f navhub
docker compose ps
```

### 2.4 首次登录

- 访问 `http://your-server:8088`
- 用户名：`superadmin`，密码：`superadmin`（首次登录后强制修改）
## 3、纯 Docker 部署

适用于已有 PostgreSQL、Redis 和 Nginx 的环境。

### 3.1 准备 config.toml

```toml
[server]
host = "0.0.0.0"
port = 8088
public_url = "https://navhub.example.com"

[database]
host = "192.168.1.100"
port = 5432
user = "navhub"
password = "your_password"
database = "navhub"

[redis]
host = "192.168.1.101"
port = 6379
password = ""
db = 0

[storage.s3]
endpoint = "https://minio.example.com"
region = "us-east-1"
bucket = "navhub"
access_key = "your_access_key"
secret_key = "your_secret_key"
```

### 3.2 启动容器

```bash
docker run -d \
  --name navhub \
  --restart unless-stopped \
  -p 8088:8088 \
  -v $(pwd)/config.toml:/app/config.toml:ro \
  ghcr.io/xjfyt/navhub:latest
```

### 3.3 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name navhub.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name navhub.example.com;

    ssl_certificate /etc/letsencrypt/live/navhub.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/navhub.example.com/privkey.pem;

    client_max_body_size 10M;

    location / {
        proxy_pass http://127.0.0.1:8088;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 4、手动构建部署

### 4.1 构建前端

```bash
cd frontend
npm ci
npm run build
```

### 4.2 构建后端

```bash
cd backend
cargo build --release
```

### 4.3 部署运行

```bash
sudo mkdir -p /opt/navhub
sudo cp backend/target/release/navhub /opt/navhub/
sudo cp -r frontend/dist /opt/navhub/frontend/
sudo cp config.toml /opt/navhub/
```

创建 systemd 服务文件 `/etc/systemd/system/navhub.service`：

```ini
[Unit]
Description=NavHub Navigation Service
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=navhub
WorkingDirectory=/opt/navhub
ExecStart=/opt/navhub/navhub
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable navhub
sudo systemctl start navhub
```

## 5、环境变量覆盖

| 变量 | 说明 |
|------|------|
| `NAVHUB__SERVER__PORT` | 服务端口 |
| `NAVHUB__SERVER__PUBLIC_URL` | 公网访问地址 |
| `NAVHUB__DATABASE__HOST` | 数据库主机 |
| `NAVHUB__DATABASE__PASSWORD` | 数据库密码 |
| `NAVHUB__REDIS__HOST` | Redis 主机 |
| `NAVHUB__STORAGE__S3__ENDPOINT` | S3 端点 |
| `NAVHUB__STORAGE__S3__ACCESS_KEY` | S3 Access Key |
| `NAVHUB__STORAGE__S3__SECRET_KEY` | S3 Secret Key |
| `NAVHUB__APP__LOG_LEVEL` | 日志级别 |

## 6、备份与恢复

```bash
# 备份 PostgreSQL
pg_dump -h <DB_HOST> -U navhub -d navhub > navhub_backup_$(date +%Y%m%d).sql

# 恢复 PostgreSQL
psql -h <DB_HOST> -U navhub -d navhub < navhub_backup_20250101.sql
```

## 7、版本升级

```bash
# docker-compose 方式
cd /opt/navhub
docker compose pull
docker compose up -d
```

## 8、安全建议

- **修改默认密码**：首次登录后立即修改 superadmin 密码
- **配置 HTTPS**：生产环境必须使用 HTTPS
- **限制端口访问**：仅通过 Nginx 暴露 443 端口
- **定期备份**：配置自动备份 PostgreSQL 数据库
- **SSO 集成**：使用 OIDC 替代密码登录
