# 五、本地开发

> **对应代码**：`dev.sh`、`backend/src/main.rs`、`frontend/vite.config.ts`
> **维护提示**：开发流程变更时同步更新本文档。

## 1、前置条件

请确保已完成 [环境准备](./环境准备.md) 中的所有安装步骤，且以下服务正在运行：

- PostgreSQL 15（端口 5432）
- Redis 7（端口 6379）

## 2、首次启动

### 2.1 克隆项目

```bash
git clone https://github.com/xjfyt/navhub.git
cd navhub
```

### 2.2 配置文件

```bash
# 复制示例配置
cp config.example.toml config.toml

# 编辑配置文件，修改数据库连接信息
# Linux / macOS
vim config.toml

# Windows（使用 VS Code）
code config.toml
```

**必须修改的配置项：**

```toml
[database]
host = "127.0.0.1"
port = 5432
user = "navhub"
password = "your_password"     # 改为你的数据库密码
database = "navhub"
```

### 2.3 安装前端依赖

```bash
cd frontend
npm install    # 或 pnpm install
cd ..
```

### 2.4 一键启动开发环境

```bash
./dev.sh
```

此脚本将并行启动：

- **后端**：`cargo watch -x run`（端口 8088，文件变更自动重编译）
- **前端**：`npm run dev`（端口 5173，Vite HMR 热更新）

### 2.5 访问应用

| 地址 | 说明 |
|------|------|
| `http://localhost:5173` | 前端开发服务器（推荐日常开发使用） |
| `http://localhost:8088` | 后端 API 服务 |

**首次登录：**

- 用户名：`superadmin`
- 密码：`superadmin`
- 首次登录后系统会强制要求修改密码

## 3、单独启动服务

### 3.1 仅启动后端

```bash
cd backend
cargo run
```

或使用 cargo-watch 实现热重载：

```bash
cargo install cargo-watch    # 首次需要安装
cargo watch -x run
```

### 3.2 仅启动前端

```bash
cd frontend
npm run dev
```

Vite 开发服务器会自动将 `/api/*` 请求代理到 `http://localhost:8088`。

## 4、日常开发流程

### 4.1 修改后端代码

1. 编辑 `backend/src/` 下的 `.rs` 文件
2. 如果使用 `cargo watch`，会自动重编译并重启
3. 手动运行时需要重新执行 `cargo run`

**代码变更自动生效的文件：**

- `backend/src/**/*.rs` → Rust 源码
- `backend/migrations/*.sql` → 数据库迁移（启动时自动执行）

### 4.2 修改前端代码

1. 编辑 `frontend/src/` 下的 `.tsx` / `.ts` 文件
2. Vite HMR 会自动刷新浏览器（无需手动刷新）
3. 修改 CSS/Tailwind 类名同样触发热更新

**代码变更自动生效的文件：**

- `frontend/src/**/*.tsx` → React 组件
- `frontend/src/**/*.ts` → TypeScript 逻辑
- `frontend/src/**/*.css` → 样式文件

### 4.3 修改配置文件

编辑 `config.toml` 后需要**重启后端**才能生效。

部分配置（如 SSO 设置）可通过超级管理员后台在线修改，无需重启。

### 4.4 新增数据库迁移

```bash
# 使用 sqlx-cli 创建新迁移
cargo install sqlx-cli --no-default-features --features rustls,postgres

cd backend
sqlx migrate add your_migration_name
```

这会在 `backend/migrations/` 目录创建新的 SQL 文件。下次启动时自动执行。

## 5、常用开发命令

### 5.1 类型检查

```bash
# 前端 TypeScript 检查
cd frontend && npm run lint

# 后端编译检查
cd backend && cargo check

# 后端 Clippy（Rust 静态分析）
cd backend && cargo clippy
```

### 5.2 数据库操作

```bash
# 连接数据库
psql -h 127.0.0.1 -p 5432 -U navhub -d navhub

# 手动执行迁移
cd backend && sqlx migrate run

# 重置数据库（谨慎！会丢失所有数据）
psql -h 127.0.0.1 -p 5432 -U navhub -d navhub -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public;"
cd backend && sqlx migrate run
```

### 5.3 Redis 操作

```bash
# 查看活跃 session
redis-cli keys 'session:*'

# 查看所有 key
redis-cli keys '*'

# 清空所有缓存（谨慎！）
redis-cli flushdb
```

### 5.4 生产构建测试

```bash
# 构建前端
cd frontend && npm run build
# 产物输出到 frontend/dist/

# 构建后端
cd backend && cargo build --release
# 二进制输出到 backend/target/release/navhub

# 运行生产版本
./backend/target/release/navhub
```

## 6、调试技巧

### 6.1 日志级别

在 `config.toml` 中调整日志级别：

```toml
[app]
log_level = "debug"    # 可选：error, warn, info, debug, trace
```

或通过环境变量：

```bash
export NAVHUB__APP__LOG_LEVEL=debug
```

### 6.2 常见问题排查

**Casdoor 登录失败：**

- 检查 `config.toml` 中的 SSO 配置是否正确
- 确认 Casdoor 应用的 `redirect_uri` 与配置一致
- 查看后端日志中的 OIDC 错误信息

**Session 失效：**

- Cookie 域名需要匹配（开发环境用 `localhost`）
- 检查 Redis 连接是否正常
- 确认 Cookie 未被浏览器阻止

**数据库连接失败：**

- 确认 PostgreSQL 服务正在运行
- 检查 `config.toml` 中的连接信息
- 确认数据库用户有足够权限

**前端请求 404：**

- 确认后端服务正在运行（端口 8088）
- 检查 Vite 代理配置是否正确
- 查看浏览器 Network 面板的请求详情

### 6.3 IDE 配置推荐

**VS Code 扩展：**

- `rust-analyzer`：Rust 语言支持
- `Tailwind CSS IntelliSense`：Tailwind 类名提示
- `ESLint`：JavaScript/TypeScript 代码检查
- `Prettier`：代码格式化

## 7、项目目录速查

| 目录 | 说明 | 修改后如何生效 |
|------|------|----------------|
| `backend/src/**/*.rs` | 后端源码 | cargo watch 自动重启 |
| `backend/migrations/*.sql` | 数据库迁移 | 下次启动自动执行 |
| `frontend/src/**/*.tsx` | 前端组件 | Vite HMR 自动刷新 |
| `frontend/src/**/*.ts` | 前端逻辑 | Vite HMR 自动刷新 |
| `config.toml` | 配置文件 | 需要重启后端 |
| `frontend/vite.config.ts` | Vite 配置 | 需要重启前端 |
