# S3 存储集成

## 概述
NavHub 支持 S3 兼容对象存储（MinIO、AWS S3 等）存储壁纸、图标等二进制资产。

## 配置

```toml
[storage.s3]
endpoint = "https://s3.example.com"
region = "us-east-1"
bucket = "navhub"
access_key = "CHANGE_ME"
secret_key = "CHANGE_ME"
path_style = true
key_prefix = "assets"
presign_ttl_secs = 86400
```

## 存储结构

```
navhub/                     # bucket
  assets/                   # key_prefix
    wallpapers/
      remote/
        {sha256}.jpg        # 远程抓取壁纸
      local/
        {sha256}.jpg        # 本地上传壁纸
    icons/
      {sha256}.png          # 图标资产
    avatars/
      {sha256}.jpg          # 用户头像
```

## 文件命名规则

- 使用 SHA-256 内容哈希作为文件名
- 相同内容的文件自动去重
- URL 泄露不会暴露其他对象

## 预签名 URL

- TTL：24 小时（presign_ttl_secs = 86400）
- 同一对象 24 小时内 URL 字节一致
- 浏览器/反代 HTTP 缓存能命中

## 公开读模式

配置公开读后，壁纸/图标 URL 不再每次重签名，CDN 才能命中缓存。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadAssets",
      "Effect": "Allow",
      "Principal": "*",
      "Action": ["s3:GetObject"],
      "Resource": ["arn:aws:s3:::navigation/assets/*"]
    }
  ]
}
```

## 本地存储回退

当未配置 S3 时，文件存储在本地 uploads/ 目录。

```toml
[app]
# 本地上传目录
upload_dir = "./uploads"
upload_max_mb = 2
```

## 文件上传限制

- 最大文件大小：2MB（可配置 upload_max_mb）
- 支持的格式：image/*
- 文件去重：SHA-256 哈希检查

## CDN 加速

配合 Cloudflare CDN 加速静态资源：
- 前端 JS/CSS：1 年缓存
- 壁纸/图标：1 个月缓存
- API/认证：不缓存

## 备份策略

### S3 存储
- 依赖桶的版本控制 + 跨区域复制
- 无需脚本备份

### 本地存储
- 使用 scripts/backup.sh 的 NAVHUB_UPLOADS_DIR 一并打包
- 定期同步到远程存储
