# 会话管理机制

## 概述
NavHub 使用 Redis 存储用户会话，实现无状态的会话管理。

## 会话存储

### Redis 键格式
`
session:{uuid} -> {user_id, role, username, email, must_change_password}
`

### TTL 设置
- 默认 TTL：7 天
- 可通过配置调整：`session_ttl_days = 7`

### 数据结构
`ust
struct SessionData {
    user_id: Uuid,
    role: String,
    username: String,
    email: String,
    must_change_password: bool,
}
`

## Cookie 配置

### Cookie 属性
`
nh_sid=xxx; HttpOnly; Secure; SameSite=Lax; Path=/
`

- **HttpOnly**：防止 XSS 攻击读取 Cookie
- **Secure**：仅 HTTPS 传输
- **SameSite=Lax**：防止 CSRF 攻击
- **Path=/**：整个域名有效

## 会话操作

### 创建会话
每次用户认证成功后生成 UUID v4 作为会话 ID，将用户信息序列化为 JSON 存储到 Redis，设置 7 天 TTL。

### 获取会话
从请求 Cookie 中提取 nh_sid，从 Redis 获取会话数据并反序列化。

### 删除会话
用户登出时从 Redis 删除会话并清除 Cookie。

## 会话中间件

### require_login
强制要求用户登录，注入 SessionUser 扩展。未登录返回 401。

### optional_login
可选登录，注入 Option<SessionUser> 扩展，用于游客可访问的路由。

## 安全特性

1. **会话固定攻击防护**：每次登录生成新的会话 ID
2. **会话劫持防护**：HttpOnly + Secure + SameSite Cookie
3. **会话过期**：自动过期 TTL 7 天 + 主动登出删除
4. **并发会话**：默认允许同一用户多个会话
5. **必须修改密码**：返回 403 Forbidden，仅允许访问修改密码端点

## 性能优化

- Redis 连接池复用连接
- JSON 序列化减少存储空间
- 支持批量删除会话
