# 权限控制设计

## 角色系统

NavHub 定义四个角色，权限从高到低：

| 角色 | 来源 | 特点 |
|---|---|---|
| superadmin | 首次启动自动创建 + 可 SSO 绑定 | 全局唯一推荐 1 个；独占 SSO 配置；可用账号密码登录 |
| admin | 由 superadmin 晋升 | 所有后台权限，无 SSO 配置入口 |
| user | 首次 SSO 登录默认 | 只能编辑自有分类；推送分类仅可改壁纸 + 侧边栏顺序 |
| guest | 预留（未登录匿名访问） | 只读 |

## 9 条权限位

| key | 说明 | superadmin | admin | user | guest |
|---|---|:-:|:-:|:-:|:-:|
| view_nav | 查看导航页 | Y | Y | Y | Y |
| use_widgets | 使用小组件 | Y | Y | Y | N |
| edit_own_nav | 编辑自有分类/图标 | Y | Y | Y | N |
| reorder_sidebar | 调整侧边栏顺序 | Y | Y | Y | N |
| edit_pushed_wallpaper | 改推送分类壁纸 | Y | Y | Y | N |
| manage_groups | 管理全局分类/推送 | Y | Y | N | N |
| manage_users | 管理用户与角色 | Y | Y | N | N |
| manage_sso | SSO 接入配置 | Y | N | N | N |
| audit_log | 查看审计日志 | Y | Y | N | N |

## 登录方式矩阵

|  | 账号密码 | Casdoor SSO |
|---|:-:|:-:|
| superadmin | Y | Y (可绑定) |
| admin / user / guest | N | Y |

超管首次 SSO 登录时，若 Casdoor 返回的 email 与 superadmin 记录匹配，则自动把 casdoor_id 绑定到超管账号。

## 推送分类（关键业务）

- groups.pushed = true, owner_id = NULL 为全局推送
- 推送分类的 icons / widgets 由后台统一维护，所有普通用户共享同一份布局
- 普通用户针对推送分类的个性化：
  - 壁纸：存于 user_preferences.pushed_group_wallpapers
  - 侧边栏顺序：存于 user_preferences.sidebar_order
- 取消推送：pushed=false，分类变成公共无主

## 后端权限实现

### 中间件注入
require_login() 注入 User { id, role } extension

### 宏检查
require_role!(admin) 拦截角色不足请求返回 403

### 细粒度检查
handler 内做所有者检查：group.owner_id == user.id 或 user.role 在 [admin, superadmin] 中

## 关键代码示例

```rust
pub fn require_at_least_admin(role: Role) -> AppResult<()> {
    if role.at_least_admin() {
        Ok(())
    } else {
        Err(AppError::Forbidden("admin_required"))
    }
}

pub fn require_superadmin(role: Role) -> AppResult<()> {
    if matches!(role, Role::Superadmin) {
        Ok(())
    } else {
        Err(AppError::Forbidden("superadmin_required"))
    }
}
```
