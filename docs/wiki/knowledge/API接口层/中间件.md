# 中间件设计

## 概述
NavHub 使用 Axum 中间件实现认证、限流、日志等横切关注点。

## 认证中间件

### require_login
强制要求用户登录，注入 SessionUser 扩展。

```rust
pub async fn require_login(
    State(state): State<Arc<AppState>>,
    mut req: Request,
    next: Next,
) -> AppResult<Response> {
    let sid = session::extract_sid(req.headers())
        .ok_or(AppError::Unauthorized)?;
    let data = session::get_session(&state, &sid)
        .await?
        .ok_or(AppError::Unauthorized)?;
    let role = Role::from_str(&data.role).unwrap_or(Role::Guest);
    
    // 检查是否必须修改密码
    if data.must_change_password && !is_must_change_password_allowed(req.uri().path()) {
        return Err(AppError::Forbidden("must_change_password"));
    }
    
    let user = SessionUser {
        id: data.user_id,
        role,
        username: data.username.clone(),
        email: data.email.clone(),
    };
    req.extensions_mut().insert(user);
    
    // 更新 last_seen_at
    bump_last_seen(state.clone(), user.id);
    
    Ok(next.run(req).await)
}
```

### optional_login
可选登录，注入 Option<SessionUser> 扩展，用于游客可访问的路由。

```rust
pub async fn optional_login(
    State(state): State<Arc<AppState>>,
    mut req: Request,
    next: Next,
) -> AppResult<Response> {
    let maybe_user: Option<SessionUser> = match session::extract_sid(req.headers()) {
        Some(sid) => match session::get_session(&state, &sid).await? {
            Some(data) => {
                // ... 构建 SessionUser
                Some(user)
            }
            None => None,
        },
        None => None,
    };
    req.extensions_mut().insert(maybe_user);
    Ok(next.run(req).await)
}
```

## 响应头中间件

### Cache-Control
全局设置 `Cache-Control: no-store`，防止缓存登录态相关响应。

```rust
Router::new()
    .layer(
        tower_http::set_header::SetResponseHeaderLayer::if_not_present(
            header::CACHE_CONTROL,
            HeaderValue::from_static("no-store"),
        ),
    )
```

## 请求体限制

### 文件上传限制
```rust
.route("/upload", post(handlers::upload::upload)
    .layer(DefaultBodyLimit::max(
        (state.cfg.app.upload_max_mb * 1024 * 1024) as usize,
    )))
```

## 健康检查

### /api/healthz
检查应用是否存活。

### /api/readyz
检查应用是否就绪（依赖 PostgreSQL 和 Redis 连接）。

## 错误处理

统一错误响应格式：
```json
{
  "error": "code_key",
  "message": "人话描述"
}
```

HTTP 状态码：
- 400：参数错误
- 401：未登录
- 403：权限不足
- 404：不存在
- 409：冲突
- 429：限流
- 500：内部错误

## 中间件顺序

1. Cache-Control 响应头
2. 路由匹配
3. 限流中间件（登录接口）
4. 认证中间件（require_login / optional_login）
5. 业务处理
