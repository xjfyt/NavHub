# 监控告警

## 概述
NavHub 提供健康检查端点和结构化日志，支持容器编排器和监控系统。

## 健康检查

### /api/healthz (存活探针)
检查应用是否存活，不检查依赖。

### /api/readyz (就绪探针)
检查应用是否就绪，包括 PostgreSQL 和 Redis 连接。

## Docker 健康检查配置
`
HEALTHCHECK --interval=30s --timeout=5s --start-period=15s --retries=3
  CMD wget -q --spider http://localhost:8088/api/readyz || exit 1
`

## 日志系统

### 日志输出
所有日志输出到标准输出 (stdout)，不进行应用层日志轮转。

### 日志收集
- systemd: 通过 journalctl 收集
- Kubernetes: 通过 Fluentd/ELK 收集
- Docker: docker compose logs 收集

### 日志格式
- pretty: 人类可读格式（开发环境）
- json: 结构化格式（生产环境）

## 审计日志

记录关键操作：
- 用户登录/登出
- 角色变更
- 分类推送/取消推送
- SSO 配置变更
- 系统设置变更

审计日志存储在 audit_log 表，保留 90 天。

## 监控指标

建议监控：
- 应用响应时间
- 错误率
- 数据库连接池使用率
- Redis 命中率
- 内存使用
- CPU 使用

## 告警规则

- 健康检查连续失败 > 3 次
- 错误率 > 5%
- 响应时间 > 2s
- 磁盘使用 > 80%
- 内存使用 > 90%
