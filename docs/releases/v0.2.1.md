# NavHub v0.2.1 发布说明

NavHub v0.2.1 是一次以壁纸可靠性和项目展示为重点的补丁版本。它解决了壁纸偶发加载缓慢、失败后长期显示主题底色、刷新后才恢复的问题，并为旧版本保存的过期对象存储签名提供自动迁移能力。

发布前安全审计同时升级了 `crossbeam-epoch 0.9.20` 与 `quinn-proto 0.11.15`，修复 `RUSTSEC-2026-0204` 和高危远程内存耗尽公告 `RUSTSEC-2026-0185`。

## 核心修复

### 1. 壁纸地址长期稳定

- 壁纸列表现在返回稳定的同源 `/uploads/...` 地址，不再把有时效的 S3 预签名 URL 暴露给前端持久化。
- 浏览器访问稳定地址时，后端即时签发新的对象存储下载地址。
- 重定向响应设置 `Cache-Control: no-store`，防止浏览器、代理或 CDN 缓存过期签名。

### 2. 旧配置自动恢复

- 启动时会识别 URL 中的 `X-Amz-Signature`、`X-Amz-Credential`、`X-Amz-Date`、`X-Amz-Expires` 等临时签名参数。
- 检测到旧地址后，根据已经保存的壁纸 UUID 调用新增的 `GET /api/wallpapers/:id` 接口，重新获取稳定地址。
- 解析成功后会自动更新访客本地偏好或已登录用户的服务端偏好，无需手动重新选择壁纸。

### 3. 加载失败与卡死恢复

- 图片真正完成加载和解码后才切换背景，不再把 `error` 当成成功。
- 对没有触发 `error` 的卡死请求增加 20 秒看门狗。
- 失败后按 1、3、8、15 秒退避重试，每次都通过稳定地址获得新的 S3 重定向。
- 解码阶段设置 8 秒保护，避免个别图片解码异常永久阻塞界面。

### 4. 预加载和随机轮换优化

- 启动预加载会先判断固定壁纸与随机轮换模式，不再让历史随机缓存覆盖当前固定壁纸。
- 缩略图保持高优先级，原图使用普通优先级，减少首屏带宽竞争。
- 随机壁纸资源池加载失败时按 5、15、30 秒退避重试。
- 不再回退到不稳定或可能受网络限制的外部直链。

## 接口变化

新增公开只读接口：

```http
GET /api/wallpapers/:id
```

接口仅返回仍处于激活状态、已经本地化且尚未过期的壁纸；其他情况返回 `404`。现有接口保持兼容。

## 升级说明

- 不包含数据库迁移。
- 不需要修改现有配置文件。
- 可以直接使用 `v0.2.1` 镜像替换旧容器。
- 旧版保存的临时壁纸 URL 会在页面启动后自动修复。

```bash
docker pull ghcr.io/xjfyt/navhub:v0.2.1
```
